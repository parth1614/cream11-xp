# Design Direction

## Goal

Build a football intelligence terminal that feels authored, sharp, and high-trust instead of looking like another generic AI landing page.

## What we are explicitly avoiding

- default Inter-plus-purple-gradient startup visuals,
- giant empty hero sections,
- over-rounded cards and vague AI copy,
- decorative motion that does not help decision-making,
- fake terminal gimmicks that hurt readability.

## Current design decisions

### 1. Editorial control-room typography

- Use `Space Grotesk` for display rhythm and hierarchy.
- Use `IBM Plex Mono` sparingly for telemetry, labels, and machine-like details.
- Keep headlines compact, assertive, and product-specific.

Why:

- Research on "AI slop" patterns highlights default typography as one of the clearest signals of generic output.
- Editorial references show that strong typography alone can create memorability without visual clutter.

### 2. Dense, useful panels instead of empty spectacle

- The homepage uses an asymmetric information layout.
- The first view surfaces recommendation framing, system state, and onboarding in parallel.
- Panels are there to answer a decision question, not fill space.

Why:

- Dashboard guidance from Smashing Magazine and Pencil & Paper emphasizes research, decluttering, and intentional data hierarchy over trend-driven mockups.

### 3. Restrained dark palette

- Use deep charcoal and warm off-white as the base.
- Use accent color only for state and emphasis.
- Avoid gradient-heavy chrome and glossy fintech clichés.

Why:

- Dark interfaces can feel premium and operational, but the design still needs contrast discipline and functional hierarchy.

### 4. Terminal attitude, not terminal cosplay

- Grid lines, monospaced labels, and telemetry panels suggest a terminal environment.
- The interface stays readable on mobile and does not require users to parse raw logs.
- Recommendation language remains clear and consumer-friendly.

Why:

- The terminal theme should speed comprehension, not become costume design.
